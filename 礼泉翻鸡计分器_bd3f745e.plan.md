# 礼泉翻鸡麻将计分器实现计划

## 1. 功能范围

- **记录 + 自动计分**：用户输入每局基础信息（胡牌者、牌型、鸡牌、缺门等），系统按规则计算每人得分
- **多局会话**：一桌多局，累计每人总分
- **持久化**：localStorage 存储会话与历史局
- **流局支持**：未听牌付 2 分/人给每位听牌玩家

## 2. 计分规则摘要（实现用）


| 项目       | 规则                                                                                                |
| -------- | ------------------------------------------------------------------------------------------------- |
| 普通胡基础分   | 2 分                                                                                               |
| 番型       | 边张/卡张 3；147/258/369 4；碰碰/清一色/七对/金钩吊 5；清七对/龙七对/青龙七对 7                                              |
| 天缺       | 有天缺者从无天缺者每人收 2 分（与胡牌无关，所有人结算）                                                                          |
| 缺门罚分     | 每张 -1 分                                                                                           |
| 鸡分       | 听牌且有鸡牌者，每张 +1 分；金鸡（1 条）×2                                                                         |
| 鸡分支付     | **每人单独结算**：有鸡牌者向**其余三家每人**各收一份鸡分（每张×倍率）                                                           |
| 流局       | 未听牌者向每位听牌者付 2 分                                                                                   |
| 点炮 vs 自摸 | **点炮**：点炮者付（基础+番型）给胡牌者；**自摸**：三家各付（基础+番型）给胡牌者，无分摊。鸡钱两种情形下均由**四人各自**按「有鸡牌者向其余三家每人收」单独结算，点炮者、其他三家都付鸡钱。 |


**鸡分支付约定**：每人单独结算。听牌且手上有鸡牌的玩家，按「鸡牌数 × 倍率（金鸡×2）」向**其余三家每人**各收一份；同时若别人有鸡牌，自己也需付给那些人。例：A 有 2 张鸡牌 → B、C、D 各付 A 2 分；B 有 1 张 → A、C、D 各付 B 1 分。

## 3. 技术方案

- **纯前端**：单页 HTML + CSS + JS，无构建工具，直接浏览器打开即可
- **结构**：单文件或 `index.html` + `style.css` + `scorekeeper.js` 分离，便于维护

## 4. 数据模型

```text
Session（会话）
├── id, createdAt, 局数
├── players: [p1, p2, p3, p4]  // 座位/昵称
└── rounds: [Round, ...]

Round（单局）
├── type: "win" | "liuju"
├── 胡牌局: winner, 点炮者(可选), 牌型[], 天缺[], 缺门罚分, 翻出牌A, 听牌者[], 每人鸡牌数[]
├── 流局: 听牌者[]
└── scores: { p1: +3, p2: -5, ... }  // 本局每人得分
```

## 5. 页面结构

- **会话创建/选择**：输入 4 人昵称，新建或从 localStorage 恢复会话
- **单局录入**：选择胡牌/流局 → 按类型展示表单（胡牌者、牌型多选、鸡牌、听牌者等）
- **结算展示**：本局每人得分 + 累计总分
- **历史局列表**：可查看、不可编辑（简化版可只展示最近 N 局）

## 6. 实现步骤

1. **搭建骨架**：HTML 结构、基础样式、localStorage 读写
2. **计分逻辑**：封装 `calcWinRound()`、`calcLiujuRound()`，严格按规则公式
3. **胡牌局表单**：缺门、番型多选、天缺勾选、翻鸡牌选择、听牌者+鸡牌数
4. **流局表单**：听牌者勾选
5. **会话与累计**：新建局、追加到 rounds、重算累计总分
6. **持久化与恢复**：保存/加载 Session，支持多会话（可选）
